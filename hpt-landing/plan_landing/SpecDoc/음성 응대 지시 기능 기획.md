# 음성 응대 지시 기능

## 개요

고객 응대 시 상담사가 음성으로 지시하면, AI가 이를 분석하여 고객에게 보낼 수 있는 정중한 응대 메시지를 자동 생성하는 기능.

상담사는 실제 고객에게 보낼 문장을 직접 말하는 것이 아니라, **AI에게 생성할 내용에 대한 지시**를 함.
예: "배송이 이틀 정도 더 걸릴 것 같다고 안내해줘" → AI가 정중한 비즈니스 메시지로 변환

## 사용자 흐름

```
[음성 응대 지시] 버튼 클릭
    → Popover 열림 + 마이크 활성화 (빨간 pulse 애니메이션)
    → 상담사 음성 입력 → STT 실시간 표시
    → [완료 (AI 생성)] 버튼 클릭
    → AI 메시지 생성 중 (파란 스피너)
    → 생성된 텍스트가 채팅 입력창에 자동 삽입
    → 상담사가 확인/수정 후 발송
```

- **취소**: 녹음 중 [음성 응대 지시] 버튼 재클릭 → STT 텍스트 폐기, Popover 닫힘
- **언어 변경**: 컴포저 헤더 우측 Globe 아이콘 클릭으로 순환 전환

### useVoiceInstruction 훅 상태

```typescript
interface UseVoiceInstructionReturn {
  isListening: boolean;        // 녹음 중 여부
  isGenerating: boolean;       // AI 생성 중 여부
  targetLanguage: VoiceLanguage; // 현재 언어 설정
  sttText: string;             // 실시간 STT 텍스트
  setTargetLanguage: (lang: VoiceLanguage) => void;
  startListening: () => void;
  stopListening: () => void;   // 취소 (텍스트 폐기)
  finishListeningAndGenerate: () => void;  // 완료 → AI 생성
  toggleListening: () => void; // 버튼 토글용
}
```

### Popover 상태 매핑

| 조건 | Popover | 버튼 색상 | 내부 표시 |
|------|---------|----------|----------|
| `!isListening && !isGenerating` | 닫힘 | secondary (회색) | — |
| `isListening` | 열림 | soft + red | "듣고 있습니다..." + STT 텍스트 + 완료 버튼 |
| `isGenerating` | 열림 | soft + blue | "응대 메시지 생성 중..." + 스피너 |

## 기술 구현 상세

### STT — Web Speech API

- `SpeechRecognition` 인스턴스는 **useEffect에서 1회만 초기화** (deps: `[]`)
- `continuous: true` — 사용자가 명시적으로 stop할 때까지 계속 인식
- `interimResults: true` — 중간 결과도 실시간 표시
- 언어(`lang`)는 `start()` 직전에 동적으로 설정
- `onend` 이벤트는 무시 — 상태 전환은 `stopListening`/`finishListeningAndGenerate` 호출부에서만 처리 (race condition 방지)
- `onGenerated`, `targetLanguage`, `sttText`는 ref로 관리하여 useEffect 재실행 방지

### AI 생성 — Claude API

- **모델**: `claude-haiku-4-5-20251001` (빠른 응답 속도 우선)
- **max_tokens**: 256
- **시스템 프롬프트**: 1~3문장 이내, 마크다운/이모지 금지, 핵심만 전달
- **프록시**: Vite dev server가 `/api/anthropic` → `https://api.anthropic.com` 중계
  - `x-api-key`, `anthropic-version` 헤더 자동 주입
  - `origin`, `referer` 헤더 제거 (CORS 인증 에러 방지)
- **API 키**: `.env` 파일의 `ANTHROPIC_API_KEY`

### 에러 처리

| 상황 | 처리 |
|------|------|
| SpeechRecognition 미지원 브라우저 | 인스턴스 null → 버튼 동작 안 함 |
| STT 에러 (마이크 거부 등) | `onerror` → `isListening = false`, 콘솔 에러 출력 |
| AI API 실패 (네트워크/인증) | catch → 입력창에 에러 메시지 삽입 |
| STT 텍스트 없이 완료 클릭 | 빈 텍스트 체크 → API 호출 스킵 |

## 브라우저 내장 Web Speech API 참고

### 장점
- **완전 무료 무제한**: 별도의 API 키나 클라우드 결제 없이 무제한 사용
- **설치 불필요**: 브라우저만 있으면 추가 설치 없이 바로 작동
- **뛰어난 인식률**: Chrome 브라우저의 경우 구글 음성 엔진 활용으로 한글 인식률 우수

### 단점 및 주의사항
- **브라우저 종속성**: Chrome/Edge는 우수, Firefox는 지원 미흡. 크로스 브라우징 제약 존재
- **네트워크 필수**: 음성 데이터를 브라우저 제조사 서버로 전송하여 변환하므로 인터넷 연결 필요
- **마이크 권한**: 최초 1회 마이크 접근 권한 팝업 허용 필요

### 적합한 환경
프로토타입, 사내 관리자용 사이트, Chrome/Edge 사용이 보장되는 컨택센터 환경에 적합.

## 음성 메모 작성 기능

### 개요

음성 응대 지시의 확장 기능으로, 상담사가 음성으로 **고객 메모** 또는 **상담 메모** 작성을 지시하면 AI가 내용을 정리하여 해당 메모 입력창에 자동 삽입한다.

- 자동 저장하지 않음 — 메모 입력창에 **출력만** 하고, 상담사가 검토 후 수동 저장
- 기존 고객 응대 메시지 생성과 동일한 음성 입력 흐름 사용

### 명령 분류 규칙

AI가 STT 텍스트를 분석하여 3가지 유형으로 분류:

| 유형 | 트리거 키워드 (예시) | AI 생성 방식 | 출력 대상 |
|------|---------------------|-------------|----------|
| `customer_message` | "~라고 안내해줘", "~라고 말해줘" | 정중한 비즈니스 메시지 생성 | 채팅 입력창 (기존) |
| `customer_memo` | "고객 메모에 적어줘", "고객 메모 작성" | 사실 기반 간결 정리 | 레퍼런스 > 고객 > 고객 메모 textarea |
| `consultation_note` | "상담 메모 적어줘", "상담 노트 작성" | 사실 기반 간결 정리 | 레퍼런스 > 컨택 > 메모 및 특이사항 textarea |

### 사용자 흐름

```
[음성 응대 지시] 버튼 클릭
    → 마이크 활성화 + STT 실시간 표시
    → 상담사: "고객 메모에 적어줘, 이 고객은 VIP 등급이고 배송 민감"
    → [완료 (AI 생성)] 클릭
    → AI가 명령 유형 분류 + 내용 정리
    → 고객 메모 입력창에 정리된 텍스트 삽입
    → 상담사가 확인/수정 후 Ctrl+Enter로 저장
```

### Fallback 처리 — 의도 분류 불가 시

다음 경우에 Popover 내 선택 모달을 표시하여 상담사가 직접 출력 대상을 선택:

- 의도가 불분명한 경우 (메모인지 응대 메시지인지 구분 안 됨)
- 두 가지 이상의 의도가 혼재된 경우 (예: "고객한테 안내하고 메모도 남겨줘")
- 고객 메모인지 상담 메모인지 구분이 안 되는 경우
- AI 응답의 JSON 파싱이 실패한 경우

```
┌─────────────────────────────────┐
│ 이 내용을 어디에서 사용하시겠습니까?  │
│                                 │
│  [고객 응대]  [고객 메모]  [상담 메모] │
│                        [취소]   │
└─────────────────────────────────┘
```

- **고객 응대** → 채팅 입력창에 삽입
- **고객 메모** → 고객 메모 textarea에 삽입
- **상담 메모** → 상담 노트 textarea에 삽입
- **취소** → 생성된 텍스트 폐기

### 데이터 흐름

```
[마이크] → Web Speech API (STT) → sttText
    → Claude API (Haiku) → JSON { type, content }
    → type 분기:
        ├─ customer_message → 채팅 입력창 (onGenerated 콜백)
        ├─ customer_memo    → CustomEvent → 고객 메모 textarea
        ├─ consultation_note → CustomEvent → 상담 노트 textarea
        └─ 파싱 실패         → Fallback 모달 → 사용자 선택 후 위 경로 중 하나
```

### AI 응답 형식

Claude API가 JSON으로 응답:

```json
{
  "type": "customer_memo",
  "content": "VIP 등급 고객. 배송 일정에 민감하여 사전 안내 필요."
}
```

- `customer_message`: 기존과 동일하게 정중한 비즈니스 메시지 생성
- `customer_memo` / `consultation_note`: 사실 기반으로 간결하게 정리 (경어체 불필요, 핵심만 기록)

### 주의사항

- 메모 입력창에 **삽입만** 수행하며, 기존 입력 중인 내용이 있으면 **덮어씀**
- 상담사가 Ctrl+Enter 또는 저장 버튼을 눌러야 실제 저장됨
- 고객 메모와 상담 메모 섹션이 화면에 보이지 않는 상태에서도 textarea 상태는 업데이트됨 (해당 탭 열면 내용 확인 가능)
