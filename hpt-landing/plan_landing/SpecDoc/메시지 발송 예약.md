# 메시지 발송 예약

## 1. 개요
- 상담방의 `sms`, `alimtalk`, `email` 채널에서 메시지를 예약 발송할 수 있다.
- `chat` 채널은 즉시 발송만 지원한다.
- 예약 메시지는 일반 채팅 이력과 분리되어 관리되며, 실제 발송이 완료된 뒤에만 채팅 이력에 노출된다.

## 2. 현재 UX
### 2.1 예약 생성
- 작성 영역의 예약 버튼은 `CalendarClock` 아이콘 버튼으로 표시된다.
- 예약 설정 UI는 모달이 아니라 버튼 바로 위에 붙는 `SchedulePickerPopover`로 열린다.
- 사용자는 달력에서 날짜를 고르고, 30분 단위 시간 목록에서 발송 시각을 선택한다.
- 오늘 날짜를 선택한 경우 현재 시각 이전 시간은 선택 목록에서 제외된다.
- 예약 팝오버 하단 액션은 `확인` 버튼만 제공한다.

### 2.2 예약 상태 표시
- 예약 시각이 설정되면 작성 영역 상단에 예약 헤더가 표시된다.
- 예약 헤더는 예약 시각을 한 줄로 보여주며, 클릭하면 같은 팝오버를 다시 연다.
- 예약 헤더 우측 닫기 버튼은 신규 예약일 때 예약 해제, 재예약 편집 중일 때 편집 취소로 동작한다.

### 2.3 템플릿 영역
- SMS/알림톡은 `TemplateArea` 내부에서 예약 헤더와 예약 버튼을 함께 사용한다.
- 템플릿 미리보기 상단 고객 정보는 반복 문장 대신 한 줄 요약으로 표시한다.
- 고객 정보가 모두 없으면 `고객 정보 없음`을 표시한다.

### 2.4 예약 목록
- 현재 방에 `scheduled`, `sending`, `failed` 상태의 예약 메시지가 1건 이상 있으면 상단에 예약 목록 버튼이 노출된다.
- 버튼 배지는 최대 `3+`까지 표시한다.
- 목록은 `ScheduledMessagesPopover`로 열리며 채널, 상태, 예약 시각, 본문 미리보기, 이메일 제목을 보여준다.
- `scheduled`, `failed` 상태의 항목은 `재예약`, `취소`가 가능하다.
- `sending` 상태는 수정과 취소를 막는다.

### 2.5 채팅방에서의 재예약
- 채팅방에 이미 연결된 예약 메시지는 일반 이력에 보이지 않는다.
- 다만 기존 예약 메시지를 재예약하는 흐름에서는 `ContactRoomArea`가 `SchedulePickerDialog`를 사용해 예약 시각을 수정한다.
- 재예약 완료 시 기존 예약 레코드를 갱신하고 방 데이터에도 반영한다.

## 3. 동작 규칙
### 3.1 예약 생성/수정
- 예약 발송 시 `useChannelComposer`가 즉시 발송 대신 `messageScheduleApi.createScheduledMessage(...)`를 호출한다.
- 재예약 편집 중 다시 저장하면 기존 예약을 취소한 뒤 새 예약을 생성한다.
- 재예약 모드 진입 시 채널 전환, 예약 시각 주입, 입력값 복원이 함께 수행된다.

### 3.2 런타임 처리
- `useScheduledMessageRuntime`가 5초 간격 polling과 `window.focus` 이벤트를 통해 발송 시각이 지난 예약을 확인한다.
- 발송 대상이 되면 상태를 `sending`으로 바꾸고 `sendMessageNow(...)`를 호출한다.
- 성공 시 `sent`, 실패 시 `failed`로 상태를 갱신한다.

### 3.3 채팅 이력 노출 규칙
- `scheduled`, `sending`, `failed`, `canceled` 상태의 예약 메시지는 일반 채팅 이력에서 숨긴다.
- `sent` 상태가 된 예약 메시지만 실제 발송 메시지처럼 채팅 이력에 표시한다.
- 채팅방 메시지 필터는 `message.deliveryMode !== 'scheduled' || message.schedule?.status === 'sent'` 규칙을 사용한다.

## 4. 데이터 모델
### 4.1 `ComposerMessage`
```ts
interface ComposerMessage {
  id: number;
  sender: 'customer' | 'agent';
  text: string;
  time: string;
  channel?: 'chat' | 'sms' | 'alimtalk' | 'email';
  subject?: string;
  templateId?: string;
  templateVariables?: Record<string, string>;
  byteLength?: number;
  status?: 'sending' | 'sent' | 'delivered' | 'failed';
  deliveryMode?: 'immediate' | 'scheduled';
  schedule?: MessageScheduleInfo;
}
```

### 4.2 `MessageScheduleInfo`
```ts
interface MessageScheduleInfo {
  scheduledAt: string;
  queuedAt: string;
  status: 'scheduled' | 'sending' | 'sent' | 'failed' | 'canceled';
  sentAt?: string;
  canceledAt?: string;
  errorMessage?: string;
}
```

### 4.3 `ScheduledMessageRecord`
```ts
interface ScheduledMessageRecord {
  id: number;
  roomId: number;
  channel: 'sms' | 'alimtalk' | 'email';
  text: string;
  subject?: string;
  templateId?: string;
  templateVariables?: Record<string, string>;
  byteLength?: number;
  recipientInfo?: string;
  deliveryMode: 'scheduled';
  schedule: MessageScheduleInfo;
}
```

## 5. 상태 전이
- 정상 발송: `scheduled -> sending -> sent`
- 취소: `scheduled -> canceled`
- 실패: `scheduled -> failed`, `sending -> failed`

## 6. 저장소와 API 추상화
- 현재 구현은 `LocalStorageMessageScheduleApi`를 사용한다.
- 저장 키는 `channelComposer_messageSchedules_v1`이다.
- 변경이 발생하면 `channel-composer-message-schedules-updated` 이벤트를 발생시켜 목록과 배지를 갱신한다.
- UI는 `IMessageScheduleApi` 인터페이스에 의존하므로 이후 서버 API 구현으로 교체 가능하다.

## 7. 관련 코드
- 작성 흐름: `src/features/channelComposer/components/ChannelComposer.tsx`
- 예약 생성/재예약 상태 관리: `src/features/channelComposer/hooks/useChannelComposer.ts`
- 예약 팝오버: `src/features/channelComposer/components/SchedulePickerPopover.tsx`
- 예약 목록: `src/features/channelComposer/components/ScheduledMessagesPopover.tsx`
- 채팅방 재예약 다이얼로그: `src/features/channelComposer/components/SchedulePickerDialog.tsx`
- 스케줄 저장소: `src/features/channelComposer/api/LocalStorageMessageScheduleApi.ts`
- 예약 런타임: `src/features/channelComposer/hooks/useScheduledMessageRuntime.ts`
- 방 메시지 동기화: `src/features/channelComposer/utils/roomMessageUtils.ts`

## 8. 정리 사항
- 더 이상 사용하지 않는 `ScheduledDraftBanner`는 제거했다.
- 사용처가 없는 `VariableBindingForm`도 제거했다.
- 예약 상태 표시는 별도 배너 컴포넌트 대신 작성 영역 상단 헤더로 통일했다.

## 9. 검증 항목
1. `sms`, `alimtalk`, `email`에서만 예약 버튼이 보인다.
2. 예약 버튼은 `CalendarClock` 아이콘으로 노출된다.
3. 예약 팝오버가 버튼 위에 열리고 날짜/시간을 같은 레이어에서 선택할 수 있다.
4. 오늘 날짜에서는 지난 시간이 시간 목록에 나타나지 않는다.
5. 예약을 설정하면 작성 영역 상단에 예약 헤더가 표시된다.
6. 예약 목록 버튼은 활성 예약이 있을 때만 보인다.
7. 예약 목록에서 `scheduled`, `failed` 항목만 `재예약`, `취소`가 가능하다.
8. 예약 메시지는 실제 발송 전까지 일반 채팅 이력에 보이지 않는다.
9. 예약 시각이 지나 런타임이 처리하면 메시지가 발송되고 채팅 이력에 추가된다.
