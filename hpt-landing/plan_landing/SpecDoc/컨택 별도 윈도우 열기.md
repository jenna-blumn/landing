## 컨택 별도 윈도우 열기 ##

### 개요

컨택을 별도 브라우저 윈도우로 열어 독립적으로 상담할 수 있는 기능. 별도 윈도우는 **탭 기반 레이아웃**(대화 탭 / 레퍼런스 탭)을 사용하여, 기존 좌우 분리 대비 공간을 효율적으로 활용한다. 사이드바에서 별도 윈도우에 열린 컨택을 클릭하면 해당 윈도우를 최상단으로 포커스한다.

### 별도 윈도우 레이아웃

기존 `ChatWorkspaceWithTabs` 컴포넌트를 재사용한다 (히스토리 모드, 스레드 모드에서 이미 사용 중).

```
[별도 윈도우 - 풀스크린]
├─ 헤더바 (플래그, 태그, VIP, 마스킹 토글, 다운로드, 새로고침)
├─ 탭: "대화" | "레퍼런스"
└─ 탭 콘텐츠
     ├─ 대화 탭 → ContactRoomArea (메시지, 컴포저, 기능바 포함)
     └─ 레퍼런스 탭 → ContactReferenceArea (정보, 컨택, 이력, 연동, 어시스턴트, 맞춤 탭)
```

- GNB, 사이드바, SharedReferenceArea 없음 — 탭 UI만 풀스크린
- `isHistoryWorkspace={false}` — 돌아가기/좋아요 버튼 미표시
- `chatContent` prop에 실제 `ContactRoomArea` 주입 (컴포저, 기능바 포함)
- `document.title`을 컨택명으로 설정 (OS 태스크바에 컨택명 표시)

### 윈도우 관리

#### 열기
- **트리거**: 대화방 헤더의 "새 창으로 열기" 버튼 (기존 `onOpenInNewWindow` prop 활용)
- **URL**: `?roomId={id}&detached=true`
- **윈도우 이름**: `_deskit_room_{roomId}` — 같은 name으로 중복 윈도우 방지
- **기본 크기**: `width=1000, height=750`

#### 포커스
- 사이드바에서 별도 윈도우에 열린 컨택 클릭 시 `windowRef.focus()` 호출
- 사용자 클릭 이벤트 내에서 호출하므로 대부분의 브라우저에서 정상 동작

#### 닫기 감지
- `BroadcastChannel('deskit-detach')`로 자식 윈도우가 `beforeunload` 시 `{ type: 'detach-closed', roomId }` 전송
- `setInterval(3초)` 폴링으로 `window.closed === true` 항목 정리 (브라우저 크래시 대비)

### 사이드바 표시

- 별도 윈도우에 열린 컨택 카드에 `Icon(['system', 'external-link'])` 아이콘 표시
- 해당 컨택 클릭 시 별도 윈도우 포커스 (메인 윈도우에서 선택하지 않음)
- 윈도우 닫힘 감지 시 아이콘 자동 제거

### Window Manager 훅 (`useDetachedWindowManager`)

```ts
interface UseDetachedWindowManager {
  openDetached: (roomId: number) => void;    // window.open() + map에 저장
  focusDetached: (roomId: number) => void;   // windowRef.focus()
  isDetached: (roomId: number) => boolean;   // !window.closed 체크
  detachedRoomIds: Set<number>;              // 반응형 Set
}
```

- `useRef<Map<number, Window>>`에 WindowProxy 참조 저장
- `useState` 카운터로 open/close 시 re-render 트리거
- 중복 열기 시 기존 윈도우 포커스

### BroadcastChannel 통신 (`useBroadcastSync`)

채널명: `deskit-detach`

| 메시지 | 발신 | 수신 | 용도 |
|--------|------|------|------|
| `{ type: 'detach-closed', roomId }` | 자식 윈도우 (beforeunload) | 메인 윈도우 | map 정리 |

- MVP에서는 `detach-closed`만 구현
- 추후 `room-updated` 메시지로 윈도우 간 데이터 동기화 가능

### App.tsx 분기

기존 `?roomId=` 파싱 로직 확장:

```ts
const isDetachedParam = params.get('detached') === 'true';
```

`isDetachedMode === true`일 때 전체 렌더트리를 단축하여 `ChatWorkspaceWithTabs`만 풀스크린 렌더링. `TaskModuleProvider`, `TeamChatModuleProvider`는 유지.

### 새 창으로 열기 버튼 위치

- `ChatRoomHeader.tsx` 우측 끝에 `iconOnly` 버튼으로 이미 배치됨 (170~179줄)
- 아이콘: `Icon(['system', 'external-link'])`, `size="sm"`, `buttonStyle="ghost"`
- 새로고침 버튼 오른쪽 (헤더 제일 오른쪽)
- `onOpenInNewWindow` prop이 전달될 때만 표시

### 브라우저 제약사항

| 항목 | 설명 |
|------|------|
| 팝업 차단 | 사용자 클릭 핸들러 안에서 `window.open()` 호출해야 차단 회피 |
| Chrome focus | 일부 버전에서 `focus()` 시 태스크바 깜빡임만 발생할 수 있음 |
| 윈도우 참조 소실 | 메인 페이지 새로고침 시 WindowProxy 참조 소실 → 새로 열기로 대응 |
| 상태 독립 | 각 윈도우가 독립적인 React 상태 보유 (mockRooms 기반이므로 동일 데이터) |

### 적용 범위

- 채팅 모드에서 사용 (일반 모드에서 "새 창으로 열기" 트리거)
- 전화 모드, 히스토리 컨택 모드, 스레드 모드에서는 미적용 (향후 확장 가능)
