# @deskit/task-module — 제품 요구사항 (PRD)

## 1. 개요

`@deskit/task-module`은 데스크잇(Deskit) 컨택센터 상담 애플리케이션을 위한 **독립형 할일 관리 모듈**이다. 상담사가 고객 응대 중 발생하는 후속 업무(SMS 발송, 콜백, 팔로업 등)를 체계적으로 관리하고, 관리자가 공지사항을 전파할 수 있는 기능을 제공한다.

## 2. 목표

- 상담 맥락과 연결된 할일 생성/관리로 업무 누락 방지
- GNB/RNB/플로팅/레퍼런스 고정 4가지 배치 모드로 기존 상담 워크플로우를 방해하지 않는 할일 관리
- 독립 모듈로 설계하여 데스크잇 외 다른 React 앱에서도 재사용 가능
- localStorage(프로토타입) / HTTP(운영) 듀얼 API 모드 지원으로 점진적 도입

## 3. 핵심 기능

### 3.1 할일 CRUD
- SMS, 콜백, 팔로업 유형의 할일 생성/수정/삭제
- 예정일·마감일 기반 상태 자동 계산 (대기/지연/완료)
- 좋아요, 고정(pin), 배경색 지정
- 서브태스크(하위 할일) 지원
- 컨택룸·메시지 연결 (말풍선-할일 연동)

### 3.2 다양한 뷰
- **드로워 (서랍 패널)**: 할일 목록, 필터 탭(전체/대기/지연/좋아요/완료/공지), 정렬
- **캘린더 뷰**: 월간 달력에 날짜별 할일 표시
- **전체 캘린더 뷰**: 확장 캘린더 + 할일 상세

### 3.3 공지사항 관리
- 관리자 전용 공지 작성 (대상 상담원 지정)
- 읽음 확인 추적 (총원/읽음/미읽음 통계)
- 공지 뷰어 (읽기 전용)

### 3.4 역할 기반 권한
| 기능 | 상담사 (consultant) | 관리자 (manager) |
|------|:---:|:---:|
| 할일 CRUD | O | O |
| 완료/좋아요/고정 | O | O |
| 공지 생성 | X | O |
| 공지 대상 지정 | X | O |
| 공지 읽음 추적 | 본인만 | 전체 |

### 3.5 컨택룸 연동
- 현재 선택된 대화방(Room)과 할일 자동 연결
- 할일에서 연결된 대화방으로 네비게이션
- 말풍선 클릭 시 AI 기반 할일 자동 생성 (목업)

## 4. 버튼 배치 모드 (buttonDisplayMode)

| 모드        | 설명                                               | 기본값 |
| ----------- | -------------------------------------------------- | :----: |
| `gnb`       | GNB(좌측 네비게이션 바) 최하단에 버튼 배치          |   O   |
| `rnb`       | RNB(우측 네비게이션 바) 최하단에 버튼 배치          |        |
| `floating`  | 화면 위 드래그 가능한 플로팅 버튼 + 드로워          |        |
| `fixed`     | 레퍼런스 패널 하단에 고정 버튼                      |        |

GNB/RNB 모드에서는 `TaskNavButton` 컴포넌트가 사용되며, 미확인 할일 뱃지와 호버 팝오버(미니 대시보드)를 제공한다. 레퍼런스 설정 패널에서 사용자가 모드를 전환할 수 있다.

## 5. 기술 스택

- React 18 + TypeScript
- Tailwind CSS (스타일링)
- lucide-react (아이콘) — 호스트 앱 DS와 독립
- localStorage / HTTP 듀얼 API
