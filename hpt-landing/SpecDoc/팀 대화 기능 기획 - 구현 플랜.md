# 팀 대화 기능 기획 리뷰 및 구체화

## Context

현재 팀 대화 기능은 사이드바에서 팀 대화방을 선택하면 **단일 대화방**이 전체 콘텐츠 영역을 대체하는 구조입니다. 기획서는 이를 **멀티 대화방 워크스페이스**로 확장하려 합니다. 기획서의 각 항목을 리뷰하고, 모호한 부분을 구체화하며, 빠진 사항을 보완합니다.

---

## 1. 기획서 항목별 리뷰

### 1-1. "팀원 선택 후 대화하기를 누르면 대화창이 뜨는 기능이 구현되지 않음"

**현재 상태 분석**: 실제로는 `CreateTeamChatDialog`에서 팀원 선택 → 대화방 생성 → `onActiveTeamRoomChange(roomId)` → `TeamChatView` 렌더링까지 **이미 구현되어 있습니다.**

**수정 제안**: 기획서의 이 항목은 "단일 대화창은 구현됨, 멀티 워크스페이스가 미구현"으로 정정해야 합니다.

---

### 1-2. "대화창은 뜨는 영역을 팀대화 워크스페이스로 지정"

**구체화**:
- 현재 `ChatModeContainer`에서 `activeTeamRoomId`가 설정되면 `TeamChatView` 하나만 렌더링
- **변경**: `activeTeamRoomId` (단일) → `openTeamRoomIds` (배열)로 확장
- 워크스페이스는 `ChatModeContainer`의 메인 콘텐츠 영역 전체를 차지
- 사이드바(팀 대화 목록)는 그대로 유지

**레이아웃 구조**:
```
┌─────────┬──────────────────────────────────────────┐
│ Sidebar │         팀대화 워크스페이스               │
│ (팀목록) │  ┌────────┬────────┬────────┐            │
│         │  │ 대화방1 │ 대화방2 │ 대화방3 │            │
│         │  │        │        │        │            │
│         │  │ [입력] │ [입력] │ [입력] │            │
│         │  └────────┴────────┴────────┘            │
└─────────┴──────────────────────────────────────────┘
```

---

### 1-3. "컬럼 선택 UI를 각 대화창 헤더에 배치 — 니가 기능 제안을 해봐"

**리뷰**: 동일한 기능(2~5열 전환)을 각 대화창 헤더마다 배치하는 것은 비효율적입니다.

**제안: 워크스페이스 툴바 방식**

```
┌──────────────────────────────────────────────────┐
│ 팀 대화 워크스페이스    [2열][3열][4열][5열]  [+]  │  ← 워크스페이스 상단 툴바
├──────────┬──────────┬──────────┬──────────────────┤
│ 대화방1  │ 대화방2  │ 대화방3  │                  │
│ 📌       │          │          │                  │
└──────────┴──────────┴──────────┴──────────────────┘
```

**근거**:
- 컬럼 수는 워크스페이스 전체에 적용되는 설정이므로 워크스페이스 레벨에 배치가 자연스러움
- 각 대화창 헤더에는 해당 대화방 고유 액션만 배치 (핀, 닫기, 대화방 이름)
- [+] 버튼으로 새 대화방 추가도 툴바에 배치

**대화창 개별 헤더 구성**:
```
┌──────────────────────────┐
│ 📌 김상담, 박대리    [×] │  ← 핀 토글 + 참여자 이름 + 닫기
├──────────────────────────┤
│ 메시지 영역              │
│                          │
├──────────────────────────┤
│ [입력창]          [전송] │
└──────────────────────────┘
```

---

### 1-4. 드래그 앤 드롭으로 위치 변경

**구체화**:
- 대화창 헤더를 드래그 핸들로 사용
- 드래그 시 시각적 피드백: 드래그 중인 대화창은 반투명 + 드롭 위치에 파란색 인디케이터
- 구현 방식: `@dnd-kit/core` 라이브러리 또는 네이티브 HTML Drag & Drop API
  - 기존 프로젝트에서 `@dnd-kit` 의존성이 없으면 네이티브 API로 가볍게 구현 권장
- 상태: `openTeamRoomIds` 배열의 순서가 곧 표시 순서

---

### 1-5. 새 대화창 추가 시 왼쪽 삽입 + 오른쪽 밀림

**구체화**:
- `openTeamRoomIds` 배열의 **앞(index 0)**에 새 ID 삽입
- 컬럼 수 초과 시: 배열 끝에서 **핀되지 않은** 대화방을 제거
  - 모든 슬롯이 핀된 경우: 추가 불가 → 사용자에게 토스트 알림 ("핀된 대화방을 해제하거나 닫은 후 추가하세요")

**예시 (3열 모드)**:
```
상태: [A, B, C] → 새 D 추가
결과: [D, A, B] (C는 핀 안 됨 → 워크스페이스에서 제거)

상태: [A(📌), B, C(📌)] → 새 D 추가
결과: [D, A(📌), C(📌)] (B는 핀 안 됨 → 제거)

상태: [A(📌), B(📌), C(📌)] → 새 D 추가
결과: 추가 불가 → 토스트: "슬롯이 모두 핀되어 있습니다"
```

---

### 1-6. 핀 기능

**구체화**:
- 핀 상태: `pinnedTeamRoomIds: Set<string>` 으로 관리
- 핀된 대화방: 헤더에 핀 아이콘 활성 표시 (채워진 핀)
- 핀 해제: 핀 아이콘 클릭 토글
- 핀된 대화방은 새 대화방 추가 시 밀려나지 않음
- 핀은 워크스페이스 내 위치와 무관 (드래그로 위치 이동 가능)

---

### 1-7. 닫기/목록 복원 동작

**구체화**:
- **닫기 (×)**: `openTeamRoomIds`에서 제거. 남은 대화방이 현재 컬럼 수 내에서 균등 분배
- **목록에서 클릭**: 이미 워크스페이스에 열려 있으면 → 해당 대화창 하이라이트 (깜빡임 효과)
- **목록에서 클릭 (미열림)**: 워크스페이스에 추가 (1-5 규칙 적용)
- 사이드바 목록에서 현재 워크스페이스에 열린 대화방은 시각적 구분 (예: 좌측 파란 바)

---

### 1-8. "스레드 대화창과 동일한 UI 사용"

**구체화**:
- 현재 `ThreadMessageList` + `ThreadComposer` 컴포넌트를 재사용
- `GroupChatRoom` 내부의 `GroupChatMessageList`/`GroupChatComposer`는 스타일이 다르므로 통일 필요
- **방안**: `ThreadMessageList`를 공통 메시지 렌더러로 사용하고, 그룹채팅 메시지를 `ChatMessage[]` 형태로 변환하여 전달
- 또는 `GroupChatMessageList`를 `ThreadMessageList` 스타일로 업그레이드

---

### 1-9. localStorage 저장 + API 확장성

**현재 상태**: 이미 `ITeamChatApi` 인터페이스 + `LocalStorageTeamChatApi` 구현 + `createTeamChatApi` 팩토리 패턴이 갖춰져 있음.

**추가 필요한 저장 항목** (워크스페이스 상태):
- `openTeamRoomIds: string[]` — 열린 대화방 목록 및 순서
- `pinnedTeamRoomIds: string[]` — 핀된 대화방 목록
- `workspaceColumns: number` — 컬럼 수 (기본값: 3)
- 저장 위치: `localStorage` key `'teamChat_workspace_state'`
- 이 상태는 UI 레이아웃 상태이므로 `ITeamChatApi`에 포함하지 않고, 별도 훅(`useTeamWorkspace`)에서 관리

## 2. 기획서에 빠진 항목 보완

### 2-1. 빈 워크스페이스 상태

팀 대화 모드 진입 시 열린 대화방이 없는 경우:
```
┌──────────────────────────────────────────┐
│         팀 대화를 시작하세요              │
│                                          │
│    사이드바에서 대화방을 선택하거나       │
│    [+ 새 대화 시작] 버튼을 클릭하세요    │
│                                          │
└──────────────────────────────────────────┘
```

### 2-2. 읽지 않은 메시지 표시

- 워크스페이스에 열려 있지 않은 대화방: 사이드바 목록에 unread 뱃지 표시
- 워크스페이스에 열려 있는 대화방: unread 뱃지 없음 (보이고 있으므로)
- 새 메시지 도착 시: 해당 대화창 하단에 "새 메시지 ↓" 스크롤 버튼

### 2-3. 반응형/최소 너비

- 각 대화창 최소 너비: 280px
- 화면 크기에 따라 자동으로 최대 컬럼 수 제한
  - < 900px: 최대 2열
  - < 1200px: 최대 3열
  - < 1500px: 최대 4열
  - >= 1500px: 5열 가능
- 사용자가 설정한 컬럼 수가 최대치를 초과하면 자동 축소

### 2-4. 워크스페이스 상태 유지

- 브라우저 새로고침 시 열린 대화방 목록, 핀 상태, 컬럼 수 복원
- localStorage에 저장 (2-1의 `teamChat_workspace_state`)

### 2-5. 키보드 접근성

- `Ctrl+숫자`: N번째 대화창으로 포커스 이동
- `Esc`: 현재 포커스된 대화창 입력 취소
- `Tab`: 대화창 간 포커스 이동

## 4. 검증 방법

1. `pnpm dev`로 개발 서버 실행
2. 사이드바에서 "팀" 탭 클릭 → 팀 대화 목록 확인
3. "새 대화방" 생성 → 워크스페이스에 대화창 표시 확인
4. 2~3개 대화방 추가 → 컬럼 레이아웃 확인
5. 툴바에서 컬럼 수 변경 → 레이아웃 반응 확인
6. 드래그 앤 드롭으로 대화창 순서 변경 확인
7. 핀 설정/해제 → 신규 추가 시 밀려나지 않음 확인
8. 대화창 닫기 → 목록에서 재오픈 확인
9. 새로고침 → 워크스페이스 상태 복원 확인
10. `npx tsc -p tsconfig.app.json --noEmit && npm run lint` 통과 확인
